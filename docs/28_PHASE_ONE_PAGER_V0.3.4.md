# StudyClaw v0.3.4 一页摘要

发布日期：`2026-03-14`

## 这是什么

`v0.3.4` 是第一阶段在 `v0.3.3` 正式基线之上的 Pad 体验补强版。

这版重点不是再扩一条新主链，而是把孩子真正每天会遇到的两个体验问题收口：

- 没有词单时，Pad 不能再把技术报错直接显示给孩子
- 做完任务时，成长鼓励要能在平板上真正播报出来

## 给家长

- 如果当天没有录入默写词单，孩子端会提示“等家长补充词单后再来默写”
- 这让家长能更快定位问题是“词单没准备”，不是“系统坏了”
- 鼓励播报会让孩子在完成任务时更容易感受到正向反馈

## 给团队

- 这次补的是正式交付体验里的高频缺口，不是边角功能
- `word_list_not_found` 已从技术异常收敛成业务等待态
- 成长鼓励现在同时具备：自动播报、手动重播、自动播报开关、真机 TTS 支持

## 给 GitHub / 对外交付

`v0.3.4` 的意义在于把 Pad 从“功能已经有了”继续推进到“孩子拿起来不会立刻撞技术墙”：

- 缺词单时是等待家长，不是系统报错
- 鼓励不是只显示文字，而是能被播报出来
- 平板真机和浏览器不再是两套完全不一致的体验

## 本版包含的关键能力

### 1. 缺词单等待态

- 识别 `word_list_not_found`
- 清理旧会话残留
- 呈现“待补充 / 重新同步”状态

### 2. 成长鼓励语音播报

- 任务板自动播报
- 语音工作台自动播报
- 手动重播
- 自动播报开关

### 3. 平板 TTS 补齐

- `dart:io` 平台启用统一 TTS
- 对插件缺失做保护
- 不让播报层故障拖垮主流程

## 验证状态

`v0.3.4` 当前已完成：

- Pad 端源码与文档同步
- `flutter analyze`
- `flutter test --no-pub`

## 一句话结论

`v0.3.4` 代表的是：孩子在 Pad 上遇到问题时，看到的是可理解的下一步；完成任务时，收到的是能真正听到的鼓励。
