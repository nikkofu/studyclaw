# StudyClaw v0.3.1 阶段一页摘要

发布日期：`2026-03-12`

## 这是什么

`v0.3.1` 是 StudyClaw 第一阶段的稳定交付版本。

这一版把三件事情真正收口到可交付状态：

- 孩子端语音助手可用
- 孩子端正向鼓励可用
- 家长端从长页面后台重构为移动优先 H5 工位

## 给家长

现在最直接的变化是家长端更像手机 App 了，而不是一张很长很长的后台页面。

- 底部固定四个主入口：`发布 / 反馈 / 积分 / 单词`
- 发布主线拆成 `范围 / 原文 / 审核 / 发布 / 拆分 / 任务 / 摘要 / 任务板`
- 手机上可以更快完成“先发作业，再看反馈，再调积分”
- 孩子完成任务和听写关键节点时，会得到更积极、更正向的反馈

对家长来说，这一版最重要的价值不是“多了一个功能”，而是日常操作终于足够顺手，适合碎片时间使用。

## 给团队

这一阶段已经形成稳定的三端协同基线：

- API 是任务、积分、词单、听写会话和统计的唯一事实源
- Parent Web 负责 `parse -> review -> confirm` 和反馈查看
- Pad App 负责任务执行、语音助手、听写推进和孩子侧鼓励

当前版本没有引入新的破坏性接口变化，重点是把已有能力从“能联调”推进到“能交付、能演示、能交接”。

这一版完成后，团队可以基于 `v0.3.1` 开启下一阶段，而不需要继续回头修补第一阶段的交付口径。

## 给 GitHub / 对外交付

`v0.3.1` 代表的不是一个实验分支，而是正式交付标签。

当前已经完成：

- 版本号同步
- release checklist 同步
- runbook / UAT / delivery readiness 同步
- 用户手册同步
- Git tag `v0.3.1`
- GitHub 远端 `main` 同步

因此它可以作为：

- 家长演示基线
- 团队协作基线
- 下一阶段正式启动前的仓库基线

## 本版包含的关键能力

### 1. 家长端移动 H5

- 单屏主流程，不再是 PC 多栏长页面
- 主屏和子页面切换更接近原生 H5
- 发布复杂作业时不再需要反复上下滚动找模块

### 2. 孩子端语音助手

- 支持“好了”“下一个”“继续”“Next”
- 支持“数学订正好了”“一课一练做完了”“全部都好了”
- 通过 STT + `/api/v1/voice-commands/resolve` 完成页面动作推理

### 3. 孩子端正向鼓励

- 单任务完成
- 分组完成
- 学科完成
- 全部完成
- 听写推进、交卷、批改完成

这些节点都会给出更积极的成长型反馈，而不是只有冷冰冰的系统提示。
