# StudyClaw v0.3.1 详细使用手册

本手册面向两类角色：

- 家长：用 Parent Web 发布作业、查看反馈、维护词单和积分
- 孩子：用 Pad App 执行当天任务、使用语音助手并完成听写练词

当前正式可发布版本是 `v0.3.1`。本版本在 `v0.3.0` 基础上继续收口了家长端移动 H5 体验：

- 家长端默认按手机 H5 工位运行，而不是 PC 仪表盘
- 发布区拆成 `范围 / 原文 / 审核 / 发布 / 拆分 / 任务 / 摘要 / 任务板` 八个子页面
- 家长端顶部首屏压缩成短头部，底部保留四大主屏导航，碎片时间更容易单手操作

`v0.3.1` 仍然保留 `v0.3.0` 已完成的两类增强能力：

- 孩子端语音助手：通过 STT + `/api/v1/voice-commands/resolve` 执行自然口令
- 孩子端正向鼓励：任务完成和听写关键节点会给出成长型反馈

本版本的核心原则仍然不变：任务、积分、词单、听写会话和统计都以 Go API 为唯一事实源。

## 1. 启动前准备

### 1.1 环境预检

```bash
bash scripts/preflight_local_env.sh
```

### 1.2 私有配置

真实密钥放在：

```text
~/.config/studyclaw/runtime.env
```

说明：

- `LLM_API_KEY` 可选；缺失时系统会回退到规则解析
- 不要把真实密钥提交进仓库

### 1.3 推荐联调地址

- API: `http://127.0.0.1:38080`
- Parent Web: `http://127.0.0.1:5173`
- Pad Web: `http://127.0.0.1:55771`

## 2. 启动系统

### 2.1 启动 API

```bash
cd apps/api-server
API_PORT=38080 go run ./cmd/studyclaw-server
```

### 2.2 启动 Parent Web

```bash
cd apps/parent-web
VITE_API_BASE_URL=http://127.0.0.1:38080 npm run dev -- --host 127.0.0.1 --port 5173
```

### 2.3 启动 Pad Web

```bash
cd apps/pad-app
flutter run -d web-server --web-hostname 127.0.0.1 --web-port 55771 \
  --dart-define=API_BASE_URL=http://127.0.0.1:38080
```

## 4. 家长端使用说明

### 4.1 先理解家长端 H5 结构

Parent Web 现在不是长页面后台，而是四个主屏：

- `发布`
- `反馈`
- `积分`
- `单词`

使用节奏是：

1. 先通过底部导航进入当前主屏
2. 再通过主屏内部的子菜单切到具体页面
3. 完成当前任务后，再切去下一个主屏

家长端重点文档可配合查看：

- [docs/PARENT_WEB_H5_MANUAL.md](/Users/admin/Documents/WORK/ai/studyclaw/docs/PARENT_WEB_H5_MANUAL.md)

### 4.2 发布每日任务

推荐按这个顺序操作：

1. 打开 Parent Web，进入底部 `发布`
2. 在 `范围` 页选择孩子与目标日期
3. 切到 `原文` 页，粘贴老师群里的原始作业文本
4. 点击“AI 解析任务”
5. 切到 `审核` 页，处理低置信度或需要人工审核的条目
6. 切到 `发布` 页，点击“确认发布选中任务”
7. 如需复查，可继续查看 `拆分 / 任务 / 摘要 / 任务板`

发布成功后：

- Pad 端能在同一天读取同一任务板
- API `/api/v1/tasks` 能返回同样的任务列表

### 4.3 查看每日反馈

在底部切到 `反馈` 后，可以继续看三个子页：

- `日报`：今天完成了多少、还剩多少
- `听写`：Pad 拍照上传后的异步批改结果
- `趋势`：周 / 月统计与鼓励方向

这些信息都来自后端：

- `/api/v1/stats/daily`
- `/api/v1/stats/weekly`
- `/api/v1/stats/monthly`
- `/api/v1/dictation-sessions`

### 4.4 积分奖惩

1. 在底部切到 `积分`
2. 进入 `记一笔`
3. 选择加分或扣分
4. 填写分值和原因
5. 提交后，Parent Web 会调用 `/api/v1/points/ledger`

如果需要复查历史记录，继续切到 `看流水`。

### 4.5 词单管理

1. 在底部切到 `单词`
2. 进入 `新建清单`
3. 填写标题、语言和绑定日期
4. 录入词项并保存
5. 如需修改，切到 `已有清单`

保存后，数据写入 `/api/v1/word-lists`，孩子端会从同一后端词单中启动听写会话。

## 5. 孩子端使用说明

### 5.1 任务执行与即时鼓励

1. 打开 Pad，确认 API 地址、家庭 ID、孩子 ID 和日期
2. 点击刷新，加载当天任务
3. 完成一项后勾选复选框
4. 系统会调用 `/api/v1/tasks/status/item`、`/group` 或 `/all` 把状态写回后端
5. 状态同步成功后，Pad 会展示一条成长型鼓励，并显示当前完成进度

孩子可能看到的鼓励示例：

- “这一步不轻松，你还是认真拿下了。”
- “现在已经完成 2/4 项，已经过半啦，继续稳稳向前。”
- “今天的挑战全部完成啦！你认真坚持到了最后，真棒。”

### 5.2 听写练词与过程鼓励

1. 进入“听写练词”页签
2. 加载当天词单
3. 点击开始，系统会调用 `/api/v1/dictation-sessions/start`
4. 后续的下一词、重播、恢复会话都基于同一个 `session_id`
5. 每推进一词、交卷、收到 AI 批改结果时，页面都会给出正向鼓励

在这个页面里，还可以直接使用语音助手：

- 说“好了”“下一个”“继续”“Next”，系统会切到下一词
- 说“重播”“再来一遍”，系统会重播当前词
- 说“上一个”，系统会返回上一词

首次使用时浏览器会申请麦克风权限，需要点允许。

### 5.3 语音助手

Pad 的“语音助手”是两段式交互：

1. 先用 STT 把孩子说的话转成 transcript
2. 再把 transcript 和当前页面上下文发给后端 `/api/v1/voice-commands/resolve`
3. 后端用规则回退 + LLM 推理返回结构化 action
4. Pad 再执行对应按钮动作

任务板场景的典型说法：

- “数学订正好了”
- “一课一练做完了”
- “全部都好了”

听写场景的典型说法：

- “好了”
- “下一个”
- “重播”

如果一句话太模糊，系统会提示“暂不执行”，不会盲目改任务状态。

## 6. 推荐交付验收故事

建议按这个顺序演示：

1. 家长端解析并确认发布当天作业
2. 孩子端刷新并勾选 1 条任务，确认出现即时鼓励
3. 孩子端说“数学订正好了”，确认语音助手能完成任务
4. 家长端刷新反馈，确认完成率同步变化
5. 家长端创建一条人工奖励积分
6. 家长端创建词单
7. 孩子端启动听写并说“好了”推进到下一词
8. 孩子端交卷并查看 AI 批改反馈
9. 家长端查看月趋势，确认任务、积分、词单、会话都被统计到

## 7. 常见问题

### 数据不一致

优先检查三个条件：

1. 三端是不是连到了同一个 API 地址
2. family / user / child / date 参数是不是一致
3. 页面是否做了刷新

### 语音助手没有反应

- 先确认浏览器已允许麦克风权限
- 再确认当前设备支持 Web Speech / STT
- 如果没有 `LLM_API_KEY`，系统仍会使用规则模式，不会完全失效

### 解析失败

- 如果没有 `LLM_API_KEY`，系统会使用规则解析
- 如果规则也无法识别，再回到家长端 `审核` 子页手动修正

### Web 构建看到 wasm warning

- 当前 `flutter build web` 会输出 `flutter_tts` 的 wasm dry-run warning
- 只要构建成功，这个 warning 在 `v0.3.1` 不是阻塞项
