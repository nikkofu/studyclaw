# StudyClaw v0.3.3 一页摘要

发布日期：`2026-03-13`

## 这是什么

`v0.3.3` 是第一阶段在 `v0.3.2` 热修复之后的正式增强版。

这版把两件事真正做成了可交付能力：

- 家长发布背诵 / 朗读任务时，系统能自动补全标准学习素材
- 孩子在 Pad 上能用连续语音完成学习动作，并拿到更有意义的背诵分析

## 给家长

- 发布古诗词 / 课文任务时，优先保留家长手动输入
- 如果家长没填，系统会先从老师原文中自动抽取标题、作者和正文
- 如果老师原文只有标题没有正文，且 LLM 可用，系统会再自动补全文本缺口
- 背诵任务默认对孩子隐藏标准原文，减少“看着读”的伪完成

## 给团队

- 这不是只改前端表单，而是打通了 `parse -> draft -> confirm -> task board -> recitation analyze`
- 解析器已修复“正文并进标题”的错误
- Pad 语音能力不再只适合几秒钟短命令，而是支持短指令、长段朗读 / 背诵、陪伴式持续监听
- 当前版本适合作为下一阶段 Agentic 学习助手能力扩展前的稳定基线

## 给 GitHub / 对外交付

`v0.3.3` 的意义在于让系统从“能记录作业”进一步走向“能理解学习内容”：

- 发布的不是只有标题的任务
- 分析的不是只有原始 transcript
- 语音不只是按钮替代，而是学习工作台入口

因此它适合作为：

- 家长演示版
- 团队协作基线版
- GitHub 正式 release 版

## 本版包含的关键能力

### 1. 学习素材自动补全

- 家长输入优先
- 老师原文自动抽取
- LLM 只补缺口
- 背诵默认隐藏标准正文

### 2. 孩子学习语音工作台

- 短指令：`好了 / 下一个 / 继续 / Next`
- 长段输入：古诗词、课文、英语朗读
- 陪伴式监听：整段学习过程中保持可用

### 3. 背诵分析

- 识别标题和作者
- 对照标准原文
- 输出逐句匹配和完成度
- 给出是否建议重背和简短建议

## 验证状态

`v0.3.3` 已完成：

- 版本号与文档同步
- API / Parent Web / Pad 核心验证
- Release checklist 与 UAT 更新
- GitHub tag 与 release 同步

## 一句话结论

`v0.3.3` 代表的是：家长少填一遍，孩子少点几次，系统开始真正拿着标准学习内容去理解孩子在学什么、读得怎么样、还差什么。
