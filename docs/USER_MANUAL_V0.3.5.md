# StudyClaw v0.3.5 使用说明

发布日期：`2026-03-14`

## 这份手册的定位

`v0.3.5` 是建立在 `v0.3.4` 正式基线之上的发布后硬化版。

对家长和孩子来说，核心使用路径并没有改变，仍然沿用 `v0.3.4` 已经稳定的体验：

1. 没有词单时，Pad 会明确告诉孩子“等家长补充”，而不是显示技术错误。
2. 成长鼓励支持语音播报，并且在平板真机上也能真正出声。
3. 家长端继续以移动 H5 工位形态完成发布、反馈、积分和单词管理。

`v0.3.5` 主要补的是交付层面的稳态：版本号、README、发布说明、操作手册和 GitHub release 页面现在保持一致。

## 给家长

### 1. 如果孩子说“默写那里进不去”

先看是不是当天没有录入词单。

从 `v0.3.5` 开始，Pad 一般会显示类似提示：

- `今天的默写词单还没准备好。先请家长补充词单，补好后再来同步就能开始默写。`

这不是系统故障，而是提示家长先回到家长端补全当天词单。

### 2. 家长应该怎么处理

推荐顺序：

1. 打开家长端 `单词`
2. 检查孩子和日期是否正确
3. 补录当天需要默写的词单
4. 回到 Pad 点 `重新同步`

如果家长已经补好词单，Pad 端重新同步后就可以直接开始默写。

### 3. 为什么不直接显示 404

因为孩子看到 `TaskApiException`、`404` 这类提示，并不能知道下一步该做什么。

`v0.3.5` 继续沿用 `v0.3.4` 的处理方式，把这类情况改成明确的业务提示，核心目的是：

- 保护孩子的学习情绪
- 让问题指向家长可执行动作
- 减少“系统坏了”的误判

## 给孩子

### 1. 默写时如果看到“待补充”是什么意思

这表示：

- 今天的词单还没有准备好
- 不是你做错了
- 先等家长补好词单，再回来开始默写

这时不用一直乱点，可以等家长补好后再点 `重新同步`。

### 2. 成长小鼓励会发生什么变化

从 `v0.3.5` 开始，成长鼓励仍然不只是显示在卡片上的一句话：

- 系统可以自动播报鼓励
- 你也可以手动点一下再听一遍
- 如果不想自动播报，也可以把自动播报关掉

### 3. 哪些地方会有鼓励播报

目前主要有两处：

1. 任务板里的 `成长小鼓励`
2. 学习语音工作台里的 `成长鼓励`

这两处都可以：

- 自动播报
- 手动重播
- 切换自动播报开 / 关

## 给演示和测试同事

### 1. 最短回归路径

Pad 端：

1. 打开 `http://127.0.0.1:55771/`
2. 切到 `听写练词`
3. 在后台不提供当天词单时，触发同步
4. 确认页面不显示 `404 / TaskApiException`
5. 确认 controller / UI 提示“默写词单还没准备好”
6. 回到任务板完成一项任务
7. 确认成长鼓励可自动播报
8. 点击 `重播鼓励`
9. 切换 `自动播报开 / 关`

### 2. 平板真机专项关注点

这版仍需要重点确认：

1. 完成任务后是否能实际播报
2. 重播按钮是否有效
3. 自动播报关闭后，是否还会静默保留文字而不自动出声

## 一句话结论

`v0.3.5` 的重点不是改孩子和家长的日常动作，而是确保他们看到的正式文档、操作手册和 GitHub release 页面都和当前真实版本一致。
