# StudyClaw 项目路线图

本文档基于 `2026-03-14` 的仓库状态更新，目标是把项目从“第一阶段已交付”推进到“第二阶段 Agentic 学习助手”。

说明：

- `v0.1.0` 到 `v0.3.5` 的第一阶段能力已经收口完成。
- 早期围绕 `v0.2.0 / v0.3.0` 的规划内容已经完成其历史使命，不再作为当前路线图主入口。
- 当前正式基线以 `README.md`、`docs/13_RELEASE_CHECKLIST.md`、`docs/17_DELIVERY_READINESS.md`、`docs/19_DELIVERY_UAT_CASES.md`、`docs/20_RELEASE_SYNC_PLAYBOOK.md` 为准。

## 1. 当前正式基线

### `v0.3.5` 第一阶段正式基线已完成

当前已经具备：

1. 家长端移动优先 H5 发布、反馈、积分、单词四大主屏
2. `parse -> review -> confirm` 的家长发布主链路
3. 孩子端任务板、任务完成、即时鼓励、每日鼓励
4. 后端统一的任务、积分、词单、听写会话、统计事实源
5. 孩子学习语音工作台，支持短口令、长段朗读 / 背诵、陪伴式持续监听
6. 背诵 / 朗读学习素材自动补全与隐藏参考原文分析
7. `preflight / smoke / demo / test / build / release checklist` 的正式交付流程

当前结论：

- `v0.3.5` 已经不是“继续补第一阶段漏洞”的临时版本，而是下一阶段启动前的正式稳定基线。

## 3. 下一目标版本

### `P0` `v0.4.0` Agentic 学习助手版

状态：`已完成并成为当前正式基线`

当前审计结论（`2026-03-16`）：

- 已满足：学习素材管理前置化、Pad 人工开始到人工结束的持续监听主路径、transcript 分段与时间点、背诵 / 朗读分析增强、家长端语音学习结果摘要复盘闭环、确定性状态 + Agent 解释的架构边界
- 当前正式状态：`v0.4.0` 已从“当前目标版本”切换为“当前正式基线”，版本号、README 和 release 资产已同步升级

目标：

把当前“能听、能记、能分析”的能力推进到真正可用的“学习语音工作台主链”。

接管治理口径：

- 核心主线固定为：`家长准备学习素材 -> 孩子语音学习 -> 系统 grounded 分析 -> 家长复盘干预`
- 全阶段坚持“确定性优先”：任务状态、积分、统计、会话事实由确定性服务维护，Agent 仅做 transcript 归一化与解释增强
- 交付顺序固定为：`SC-01 API 契约` -> `SC-03 Pad 主链` -> `SC-04 Parent 复盘` -> `SC-05 集成收口`，`SC-02 Agent` 全程并行支撑
- 协作组织固定为五 lane：`SC-01-GO-API`、`SC-02-GO-AGENT`、`SC-03-FLUTTER-PAD`、`SC-04-PARENT-WEB`、`SC-05-INTEGRATION`

`v0.4.0` 必须覆盖这 6 类能力：

1. 家长在发布任务时就能稳定管理学习素材，不再让孩子临场补录
2. 孩子端语音工作台支持“人工开始到人工结束”的持续监听主路径
3. 语音内容能形成接近会议纪要式的分段 transcript 时间线
4. 背诵 / 朗读分析能对 noisy transcript 做标题识别、正文对齐、错漏定位和重试建议
5. 家长端可查看孩子语音学习结果摘要，而不只是任务是否完成
6. 所有语音 / 背诵结果依然遵循“确定性状态 + Agent 解释”的架构边界

版本重点：

- 学习素材管理前置化
- 语音陪伴与持续监听
- transcript 分段和归档
- 背诵 / 朗读对齐分析增强
- 家长复盘与反馈可视化

## 4. 当前明确后置的能力

这些能力不进入 `v0.4.x`：

- 多智能体编排
- 强实时推送
- 自动奖惩决策
- 复杂宠物养成 / 重游戏化系统
- 作文自动批改大闭环
- 复杂家庭知识图谱

原因：

当前最重要的是先把下面这个核心价值做稳：

“家长布置并准备素材 -> 孩子通过语音完成学习任务 -> 系统做 grounded 分析 -> 家长看到可复盘结果”

## 6. `v0.4.0` 成功标准

`v0.4.0` 是否达标，用下面条件判断：

1. 家长发布背诵 / 朗读任务时，学习素材来源和隐藏策略清晰可控
2. 孩子在语音工作台里能手动开始、持续说话、手动结束，不被几秒无声自动打断主流程
3. 系统能给出可追溯的 transcript 分段与时间点
4. 背诵 / 朗读分析能输出标题 / 作者 / 完成度 / 逐句匹配 / 错漏点 / 是否建议重试
5. 家长端能看到孩子某次语音学习的结果摘要
6. `preflight / smoke / demo / go test / web test-build / pad analyze-test-build` 全部通过

## 7. 推荐节奏

建议按下面顺序推进：

1. 先锁定学习素材与语音会话契约
2. 再补齐 Pad 学习语音工作台主链
3. 然后补齐家长端学习素材与语音复盘
4. 最后统一做评测、演示和发版收口

如果继续沿用旧的 `v0.2.0` 派单口径，会导致团队把已经完成的问题重复做一遍，而不是把下一阶段真正关键的“学习语音工作台”做深做稳。
