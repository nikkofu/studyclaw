# StudyClaw 项目路线图

本文档基于 `2026-03-09` 的仓库状态更新，反映当前真实进度和下一步优先级。

## 1. 当前里程碑

### `v0.1.0` 已交付

`v0.1.0` 的目标是验证最小闭环，目前已完成：

1. 家长端粘贴学校群原始任务文本
2. Go 后端中的解析模块做 `LLM 优先 + 规则兜底` 解析
3. 家长审核并确认创建任务
4. 任务按 `学科 -> 作业分组 -> 原子任务` 写入 Markdown
5. Pad 端拉取任务板并同步完成状态

### `v0.1.1` 联调基线已交付

`v0.1.1` 的目标是把 MVP 从“能单独开发”推进到“可稳定联调”，目前已完成：

1. Go API 错误契约统一
2. Parser / weekly insight 回归测试增强
3. Parent Web 按日期创建任务与自动化测试
4. Pad 端分层、同步测试与 Web 构建验证
5. 本地 `preflight` / `smoke` 基础脚本与派单文档

## 2. 接下来版本优先级

### `P0` `v0.2.0` 演示稳定版

目标：把当前联调基线变成稳定可演示、可重复验收的内部版本。

- 把 preflight / smoke 扩展成更完整的演示检查流
- 补全家长端和 Pad 端的真实后端联调回归
- 增加更多学校群任务样本测试与质量回归说明
- 固化 API 契约、错误码和 smoke 输入样例
- 增加最小发布清单和演示环境说明

### `P1` `v0.3.0` 任务理解增强版

目标：让 AI 对“续做、订正、复习、条件任务”理解更稳定。

- 引入历史任务上下文
- 做重复任务、冲突任务、续做任务识别
- 增加低置信度任务确认机制
- 优化周报摘要质量

### `P2` `v0.4.0` 多模态输入版

目标：支持家长上传图片作业或黑板照片。

- 接入 OCR / VLM
- 图片 + 文本混合解析
- 图片解析样本回归集

### `P3` `v0.5.0` 成长反馈版

目标：补齐成长记录和更完整的家长反馈。

- 周报成品页
- 基础积分结算规则
- 家长端统计视图

## 3. 明确后置的能力

这些内容暂不进入近两版：

- 情绪识别
- 传感器数据接入
- WebSocket 实时聊天
- 长期记忆 / 向量数据库
- 强任务调度或 AI 替孩子排序

原因很明确：当前产品更需要先证明“学校群任务自动转为可执行任务板”这个核心价值。

## 4. 近两周执行建议

### 第 1 周

- 完成 `v0.1.1` 版本冻结与发布记录
- 补 10 到 20 组真实学校群任务解析样本
- 固化家长端和 Pad 端真实后端联调清单
- 整理演示流程、运行文档和版本说明

### 第 2 周

- 增加解析质量对比数据
- 增加低置信度任务标注与家长确认体验
- 优化 Pad 端任务板的细节交互
- 输出首轮内部试用反馈表

## 5. 成功标准

`v0.2.0` 是否成功，用这几条衡量：

1. 新家长第一次看文档，30 分钟内可以跑起完整演示
2. 学校群式任务文本，大部分样本能自动拆成学科和原子任务
3. 孩子可以独立在 Pad 端完成勾选，不需要家长手把手操作
4. 家长可以随时看到最新进度和当天 Markdown 记录
5. 团队可以通过脚本和派单文档完成下一轮并行协作
