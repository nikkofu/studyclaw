# StudyClaw 项目开发计划与路线图 (Roadmap)

本项目采用敏捷开发模式，划分为四个主要阶段。每个阶段力求产出可运行的版本（MVP -> Alpha -> Beta -> v1.0）。

## 阶段一：奠基与核心链路跑通 (MVP)
**目标:** 验证“服务端 <-> Agent Core <-> LLM”的核心链路，实现最基础的文本布置与展示。
**时间预期:** 2-3 周

1. **环境与基础设施搭建**
   - 初始化 Monorepo，配置 ESLint、Prettier。
   - 搭建 API Server 基础架构（Auth, DB 联调）。
   - 搭建 Agent Core（Python Flask/FastAPI 环境，接入指定 LLM Provider）。
2. **核心业务 API 开发**
   - 用户与家庭组的建立（Mock 登录即可）。
   - 任务（Task）数据结构的 CRUD 接口。
3. **Agent 核心链路开发**
   - 编写首个 Prompt Template（作业解析：提取科目、任务名称）。
   - 实现简单的文本输入 -> LLM 提取 -> API 存入数据库的流程。
4. **客户端雏形**
   - Pad App 静态页面搭建（首页大看板）。
   - Parent Web 管理页面静态搭建（任务列表）。

## 阶段二：语音交互与多模态赋能 (Alpha)
**目标:** 引入 TTS/ASR，实现交互式听写播报与作业图片解析。
**时间预期:** 3-4 周

1. **听写助手模块开发（Agent Core / API）**
   - 接入 Edge TTS 或其他云端服务实现单词连贯播报能力。
   - 在 Agent 内加入“听写出题”及“判决”逻辑。
2. **多模态解析（Agent Core）**
   - 接入 GPT-4V/Claude 3.5 Sonnet 等视觉大模型能力，支持家长拍摄作业黑板图片直接解析。
3. **Pad 端多媒体开发**
   - 接入录音接口与文件上传能力。
   - 实现“点击播放下一词”的闭环交互 UI。
4. **管理端端功能扩展**
   - 图片上传模块开发。
   - 听写单词表管理界面。

## 阶段三：成长体系与 WebSocket 实时能力 (Beta)
**目标:** 引入奖励机制，打通双端的即时通信。
**时间预期:** 3 周

1. **实时消息推送**
   - 部署 WebSocket 服务。
   - 建立“亲子互动频道”（家长留言推送至 Pad 并在界面弹出提示）。
2. **积分打卡与成长系统**
   - 数据库积分表设计。
   - 完成指定任务自动发放 Claw 积分。
   - Pad 端动效：虚拟宠物或进度徽章更新。
3. **应用商店审核材料准备**
   - 合规性审查、隐私政策撰写、应用商店宣传图设计。

## 阶段四：打磨与用户测试 (V1.0)
**目标:** 提供给种子用户使用，收集反馈并优化体验。
**时间预期:** 持续进行

1. **内测与错误追踪**
   - 接入 Sentry 或类似监控工具。
   - 邀请家庭用户进行内测。
2. **性能与体验优化**
   - Padding 和动效调优。
   - LLM 响应时延（TTFT - Time To First Token）的优化与本地缓存。
3. **准备 V1.0 发布**及后续版本（如错题本、番茄钟功能开发）的规划。
