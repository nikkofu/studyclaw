# StudyClaw v0.3.3 使用说明

发布日期：`2026-03-13`

## 这份手册的定位

`v0.3.3` 是建立在 `v0.3.2` 热修复版之上的正式能力增强版。

这次不是单点修 bug，而是把“孩子学习语音工作台”和“学习素材自动补全”真正纳入三端主链：

1. 家长发布背诵 / 朗读任务时，不必再手工重复录入标题、作者和正文。
2. 孩子在 Pad 上进行短指令、长段朗读 / 背诵和陪伴式学习时，都能使用连续监听。
3. 背诵分析可以结合隐藏参考原文，对 noisy transcript 做更可靠的完成度判断。

## 给家长

### 1. 发布背诵 / 朗读任务的推荐顺序

在家长端底部切到 `发布` 主屏后：

1. 在 `范围` 页选择孩子和日期。
2. 进入 `原文` 子页面，粘贴老师消息。
3. 点击 `AI 解析任务`。
4. 进入 `审核` 子页面，检查学习素材字段。
5. 确认无误后发布。

如果老师原文里包含古诗词或课文正文，系统通常会自动带出：

- `reference_title`
- `reference_author`
- `reference_text`
- `analysis_mode`

### 2. 学习素材自动补全的优先级

系统按下面顺序决定最终使用哪份学习素材：

1. 家长手动输入
2. 老师原文自动抽取
3. LLM 补全剩余缺口

这意味着：

- 家长已经填过的内容，不会被 LLM 反向覆盖
- 老师原文足够完整时，不必额外依赖 LLM
- 只有在原文缺正文、但标题明确的情况下，才让 LLM 参与补全

### 3. 审核阶段应该重点看什么

对背诵 / 朗读任务，重点核对这四项：

1. 任务标题是否准确
2. 标题和作者是否匹配
3. 正文是否完整
4. `analysis_mode` 是否合理

常见判断：

- 古诗词通常是 `classical_poem`
- 课文背诵通常是 `text_recitation`
- 英语朗读通常是 `english_reading`
- 普通朗读通常是 `read_aloud`

### 4. 为什么背诵任务会默认隐藏正文

`v0.3.3` 默认对 `背诵` 任务设置 `hide_reference_from_child=true`。

原因很直接：

- 如果孩子在 Pad 上直接看到标准正文，就无法判断他是在背，还是在照着读
- 隐藏正文不影响系统分析，因为后台仍然保留参考原文
- 家长仍然可以在审核页修改和查看这份标准正文

如果你的使用场景更像“跟读”而不是“背诵”，可以把任务改成朗读类任务。

## 给孩子

### 1. 什么是“孩子学习语音工作台”

它不是只会识别一个“好了”的按钮替代器，而是一套更适合学习场景的语音输入方式。

当前支持三类模式：

1. 短指令模式
2. 长段朗读 / 背诵模式
3. 陪伴式持续监听模式

### 2. 短指令模式怎么用

适合这些场景：

- 单词默写做完一个，想说“下一个”
- 一课一练做完了，想说“好了”
- 数学订正完成，想说“数学订正好了”

推荐口令：

- `好了`
- `下一个`
- `继续`
- `Next`
- `数学订正好了`
- `一课一练做完了`

### 3. 长段朗读 / 背诵模式怎么用

适合这些场景：

- 读背古诗词
- 背课文
- 英语朗读
- 对话练习
- 口头日记

特点：

- 不是几秒没说话就立刻结束
- 会把语音按停顿逐段记录
- 可以把多段 transcript 汇总给分析器

### 4. 陪伴式持续监听适合什么

适合一段较长的学习过程，例如：

- 单词默写整轮练习
- 连续背诵多段古诗或课文
- 跟读后接纠音和分析

这种模式更像“学习陪伴中保持在线”，直到人工结束监听。

## 背诵分析怎么理解

### 1. 分析器会做什么

当系统拿到 transcript 和标准原文后，会尽量输出：

- 识别出的标题
- 识别出的作者
- 对照后的标准正文
- 每一句的匹配情况
- 完成度比例
- 是否建议重背
- 简短总结和建议

### 2. 为什么识别出来的字可能不准，但分析仍然有价值

STT 在古诗词、同音字和儿童发音场景下，原始转写经常会很乱。

例如：

- `黄师塔前` 可能被识别成 `黄思塔前`
- `懒困倚微风` 可能被识别成 `缆会以微风`

`v0.3.3` 的目标不是要求 STT 一字不差，而是让系统在有标准原文时，仍然能推断：

- 孩子大体是不是在背这首诗
- 哪几句基本对上了
- 哪些地方明显不稳

### 3. 什么时候建议重背

系统通常会在下面情况建议再背一遍：

- 标题和正文主体没有对上
- 某几句匹配率明显偏低
- 只背出了前半段
- 中途停顿很多，内容缺失明显

## 给演示和测试同事

### 1. 最短回归路径

家长端：

1. 打开 `http://127.0.0.1:5173/`
2. 进入 `发布`
3. 选择孩子和日期
4. 粘贴包含古诗词正文的老师消息
5. 点击 `AI 解析任务`
6. 在 `审核` 子页面确认学习素材字段已经自动带出

孩子端：

1. 打开 `http://127.0.0.1:55771/`
2. 进入任务板
3. 点击“开始说话”
4. 说“好了”或“数学订正好了”
5. 再进入背诵类任务，说出一段古诗词
6. 确认系统能输出完成度和建议，而不是只保留原始 transcript

### 2. 本版重点不变项

- API 仍然是三端唯一事实源
- 家长端仍保持移动 H5 工位
- 正向鼓励仍在任务完成、听写推进和批改结束时触发

## 一句话结论

`v0.3.3` 把“发布背诵 / 朗读任务”和“孩子真实用语音完成学习动作”两条链路真正接了起来：
家长少填表，孩子少点按钮，系统能拿着标准原文做更可靠的学习分析。
