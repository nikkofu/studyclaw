# StudyClaw v0.2.0 详细使用手册

欢迎使用 StudyClaw 第一阶段正式签收版。本手册旨在指导家长（管理端）与孩子（执行端）如何高效协同，利用 AI 简化每日学习管理。

## 1. 快速启动指南

在开始使用前，请确保您已完成以下本地环境准备：

1. **环境预检**：执行 `bash scripts/preflight_local_env.sh`。
2. **私有配置**：确保 `~/.config/studyclaw/runtime.env` 中已配置 `LLM_API_KEY`。

### 1.1 启动后端服务 (Go)
```bash
cd apps/api-server
go run ./cmd/studyclaw-server
```
- **默认地址**：`http://localhost:8080`
- **健康检查**：访问 `http://localhost:8080/ping` 看到 `pong` 表示启动成功。

### 1.2 启动家长管理端 (React)
```bash
cd apps/parent-web
npm install  # 仅首次运行需要
npm run dev
```
- **默认地址**：`http://localhost:5173`
- **注意**：若需要修改后端地址，请调整根目录或该目录下的 `.env` 文件中的 `VITE_API_BASE_URL`。

### 1.3 启动 Pad 执行端 (Flutter)
```bash
cd apps/pad-app
flutter pub get  # 仅首次运行需要
# 在浏览器(Chrome)中运行，并指定后端 API 地址
flutter run -d chrome --dart-define=API_BASE_URL=http://localhost:8080
```
- **默认地址**：通常为 `http://localhost:8080` (注意端口可能与 Go 后端冲突，Flutter 会自动递增，请以终端输出为准)。
- **推荐**：开发联调建议在 Chrome 中运行，方便查看后端请求日志。

## 2. 家长端 (Parent Web) 使用说明

### 2.1 发布每日任务
1. **进入发布页**：选择孩子头像及目标日期（建议默认当日）。
2. **粘贴原文**：将学校微信群/QQ群里的老师作业原封不动粘贴到输入框。
3. **AI 解析**：点击“AI 解析任务”。系统会自动按学科、分组拆解任务。
4. **审核草稿**：
   - **置信度预警**：置信度低于 0.7 的任务会标红，请重点核对。
   - **风险提示**：AI 会自动识别“续做”、“订正”等可能需要家长介入的模糊任务。
5. **确认发布**：点击“确认发布”，任务将正式同步到后端，并可在 Pad 端可见。

### 2.2 积分奖惩管理
1. **手工奖惩**：在“积分管理”区域，输入分值（支持负数扣分）和原因。
2. **查看流水**：系统会实时展示来自后端的积分明细，不再依赖本地缓存。
3. **自动积分**：孩子在 Pad 端每完成一项任务，系统会根据后端规则自动发放奖励分。

### 2.3 单词清单创建
1. **新建清单**：输入清单标题（如“Unit 3 词汇默写”），选择语言（英语/语文）。
2. **录入词项**：每行一个单词或词组。
3. **保存同步**：点击“创建单词清单”，数据将持久化到后端。

---

## 3. 孩子端 (Pad App) 使用说明

### 3.1 任务执行
1. **同步任务**：打开 Pad，选择日期，点击刷新。
2. **勾选完成**：孩子完成一项作业后，点击左侧复选框。状态会实时 PATCH 到后端。
3. **查看简报**：点击“今日简报”，查看当前进度、完成率以及 AI 生成的鼓励语。

### 3.2 单词听写播放
1. **加载清单**：在单词卡片区域，点击“加载当日清单”。
2. **开启听写**：点击“开始听写”，进入后端驱动的 `dictation-session`。
3. **交互控制**：
   - **播放**：自动朗读当前单词。
   - **下一个**：确认掌握后跳至下一个，进度由后端记录，刷新不丢失。
   - **重播**：点击重播当前词。

---

## 4. 数据统计与 AI 观察
1. **周趋势**：展示近 7 天的任务完成率与积分变化曲线。
2. **月度分析**：通过 `/api/v1/stats/monthly` 获取按周聚合的统计数据，客观反映学习趋势。
3. **AI 鼓励**：AI 会基于真实统计值（而非前端估算）输出正向的心理学激励建议。

---

## 5. 常见问题 (FAQ)
- **数据不一致？** 请点击各页面的“刷新”按钮，确保拉取的是后端最新事实源。
- **解析失败？** 检查网络连接及 `LLM_API_KEY` 是否有效。
- **积分未更新？** 积分展示已切为权威余额模式，请确认后端 `points/ledger` 是否已成功写入。
