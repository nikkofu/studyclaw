# StudyClaw v0.2.0 详细使用手册

归档说明：

- 本手册保留为 `v0.2.0` 历史正式版本的操作说明。
- 当前可发布增强版请使用 `docs/USER_MANUAL_V0.3.1.md`。

本手册面向两类角色：

- 家长：用 Parent Web 发布作业、查看反馈、维护词单和积分
- 孩子：用 Pad App 执行当天任务和听写练词

当前正式交付版本是 `v0.2.0`。本版本的核心原则是：任务、积分、词单和统计都以 Go API 为唯一事实源。

## 1. 启动前准备

### 1.1 环境预检

```bash
bash scripts/preflight_local_env.sh
```

### 1.2 私有配置

真实密钥放在：

```text
~/.config/studyclaw/runtime.env
```

说明：

- `LLM_API_KEY` 可选；缺失时系统会回退到规则解析
- 不要把真实密钥提交进仓库

### 1.3 推荐联调地址

- API: `http://127.0.0.1:38080`
- Parent Web: `http://127.0.0.1:5173`
- Pad Web: `http://127.0.0.1:55771`

## 2. 启动系统

### 2.1 启动 API

```bash
cd apps/api-server
API_PORT=38080 go run ./cmd/studyclaw-server
```

### 2.2 启动 Parent Web

```bash
cd apps/parent-web
VITE_API_BASE_URL=http://127.0.0.1:38080 npm run dev -- --host 127.0.0.1 --port 5173
```

### 2.3 启动 Pad Web

```bash
cd apps/pad-app
flutter run -d web-server --web-hostname 127.0.0.1 --web-port 55771 \
  --dart-define=API_BASE_URL=http://127.0.0.1:38080
```

## 3. 推荐演示数据

为了避免“今天 / 明天”造成混淆，建议固定使用：

- `family_id=306`
- `user_id / child_id=1`
- `assigned_date=2026-03-12`

## 4. 家长端使用说明

### 4.1 发布每日任务

1. 打开 Parent Web，选择孩子与目标日期。
2. 把老师群里的原始作业文本粘贴到输入框。
3. 点击“AI 解析任务”。
4. 检查系统给出的学科、分组和原子任务。
5. 重点确认低置信度或需要人工审核的条目。
6. 点击“确认发布选中任务”，把任务写入后端。

发布成功后：

- Pad 端能在同一天读取同一任务板
- API `/api/v1/tasks` 能返回同样的任务列表

### 4.2 查看每日反馈

家长端可以查看：

- 当日完成数和完成率
- 当日积分增减
- 周趋势和月趋势
- 词单与听写批改结果

这些信息都来自后端：

- `/api/v1/stats/daily`
- `/api/v1/stats/weekly`
- `/api/v1/stats/monthly`
- `/api/v1/dictation-sessions`

### 4.3 积分奖惩

1. 在“积分”区域输入分值和原因。
2. 提交后，Parent Web 会调用 `/api/v1/points/ledger`。
3. 提交成功后，家长端和 Pad 端看到的是同一份余额结果。

### 4.4 词单管理

1. 在“单词清单”区域输入标题、语言和词项。
2. 需要时可先调用词单解析功能做预处理。
3. 保存后，数据写入 `/api/v1/word-lists`。
4. 孩子端会从同一后端词单中启动听写会话。

## 5. 孩子端使用说明

### 5.1 任务执行

1. 打开 Pad，确认 API 地址、家庭 ID、孩子 ID 和日期。
2. 点击刷新，加载当天任务。
3. 完成一项后勾选复选框。
4. 系统会调用 `/api/v1/tasks/status/item`、`/group` 或 `/all` 把状态写回后端。

每次完成任务后，Pad 会立刻给出一条积极的成长型鼓励，并结合当天已完成进度提示孩子继续向前。例如：

- “这一步不轻松，你还是认真拿下了。”
- “现在已经完成 2/4 项，离全部完成只差最后一步。”

孩子端看到的“今日积分”和家长端看到的积分余额是同一份后端结果，不再依赖前端估算。

### 5.2 听写练词

1. 进入“听写练词”页签。
2. 加载当天词单。
3. 点击开始，系统会调用 `/api/v1/dictation-sessions/start`。
4. 后续的下一词、重播、恢复会话都基于同一个 `session_id`。
5. 每推进一词、交卷、收到 AI 批改结果时，页面都会给出正向鼓励，避免把反馈做成冷冰冰的系统提示。

在这个页面里，还可以直接使用语音助手：

- 说“好了”“下一个”“继续”“Next”，系统会切到下一词
- 说“重播”“再来一遍”，系统会重播当前词
- 说“上一个”，系统会返回上一词

首次使用时浏览器会申请麦克风权限，需要点允许。

### 5.3 语音助手

Pad 的“语音助手”不是简单的关键词映射，而是两段式交互：

1. 先用 STT 把孩子说的话转成 transcript
2. 再把 transcript 和当前页面上下文发给后端 `/api/v1/voice-commands/resolve`
3. 后端用规则回退 + LLM 推理返回结构化 action
4. Pad 再执行对应按钮动作

任务板场景的典型说法：

- “数学订正好了”
- “一课一练做完了”
- “全部都好了”

听写场景的典型说法：

- “好了”
- “下一个”
- “重播”

如果一句话太模糊，系统会提示“暂不执行”，不会盲目改任务状态。

## 6. 推荐交付验收故事

建议按这个顺序演示：

1. 家长端解析并确认发布当天作业
2. 孩子端刷新并勾选 1 条任务
3. 家长端刷新反馈，确认完成率同步变化
4. 家长端创建一条人工奖励积分
5. 家长端创建词单
6. 孩子端启动听写并推进到下一词
7. 家长端查看月趋势，确认任务、积分、词单、会话都被统计到

## 7. 常见问题

### 数据不一致

优先检查三个条件：

1. 三端是不是连到了同一个 API 地址
2. family / user / child / date 参数是不是一致
3. 页面是否做了刷新

### 解析失败

- 如果没有 `LLM_API_KEY`，系统会使用规则解析
- 如果规则也无法识别，再回到家长端手动审核和修正

### Web 构建看到 wasm warning

- 当前 `flutter build web` 会输出 `flutter_tts` 的 wasm dry-run warning
- 只要构建成功，这个 warning 在 `v0.2.0` 不是阻塞项
