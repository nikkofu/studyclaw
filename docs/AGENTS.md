## Integration Group

This subtree is owned by the integration and release group.

### Owned scope
- `docs/`
- Cross-team planning, runbooks, release notes, coordination docs

### Goal
- Keep the project runnable, understandable, and easy to hand off across teams

### Do not do
- Do not implement business logic here
- Do not silently change source-code behavior from documentation tasks

### Default policy
- Prefer documenting the current truth of the codebase
- Keep runbooks and security guidance aligned with the actual runtime architecture
