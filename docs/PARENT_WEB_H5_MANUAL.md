# StudyClaw 家长端移动 H5 操作手册

本文档用于 `v0.3.4` 正式版本中的家长端移动 H5 专项操作说明。它与主用户手册配套使用，重点解释家长端从长页面重构为多主屏、多子页面之后的使用方式，以及背诵 / 朗读类任务的学习素材自动补全。

## 1. 适用范围

- 入口地址：`http://127.0.0.1:5173`
- 适用设备：
  - 手机浏览器
  - 桌面浏览器访问时的“手机 H5 工位”形态
- 适用能力：
  - 发布作业
  - 查看反馈
  - 积分奖惩
  - 单词清单管理

## 2. 页面结构

家长端现在固定为手机工作流，而不是 PC 仪表盘。

从上到下分成五层：

1. 顶部摘要区
2. 当前主屏内容
3. 主屏内的二级菜单
4. 二级菜单对应的子页面
5. 底部固定导航与阶段性动作条

核心设计原则：

- 一只手能完成高频操作
- 先处理当天任务，再切到反馈 / 积分 / 单词
- 当前主屏永远置顶，避免在长页面里反复寻找区域
- 关键操作默认一次看全，不要求左右滑动去找完整内容
- 复杂区域优先拆成子页面，用菜单切换和滑动过渡代替长表单堆叠

## 3. 顶部摘要区怎么用

顶部会显示：

- 当前日期
- 当前主屏摘要
- 今日工作摘要

为了避免手机首屏过长，顶部摘要已经压缩成短头部：

- 不再把 PC 式多块大统计卡放在最上面
- 先让家长知道“现在在哪个主屏”
- 再决定是否立刻进入发布、反馈、积分或单词

建议家长先看这里，再决定先处理哪件事：

- 如果还没布置作业，直接进入“发布”
- 如果孩子已经在做题，先去“反馈”
- 如果今天有明显表现，去“积分”
- 如果晚上要默写，去“单词”

## 4. 底部导航说明

底部固定 4 个入口：

- `发布`
- `反馈`
- `积分`
- `单词`

点击任意入口后：

- 对应功能区会切成当前主屏
- 当前主屏会被置顶
- 底部导航保留，不需要来回滚动查找入口

进入主屏后，核心内容会继续按二级菜单切换：

- 发布：`范围 / 原文 / 审核 / 发布 / 拆分 / 任务 / 摘要 / 任务板`
- 反馈：`日报 / 听写 / 趋势`
- 积分：`记一笔 / 看流水`
- 单词：`新建清单 / 已有清单`

这些二级菜单不是简单展开收起，而是子页面切换：

- 点击后会直接切到对应子页面
- 子页面之间带左右滑入滑出动效
- 当前子页面保留数据，不会因为切走就清空

## 5. 日常推荐操作路径

### 5.1 放学后先布置作业

1. 进入 `发布`
2. 选孩子、日期
3. 粘贴老师群消息
4. 点击 `AI 解析任务`
5. 审核风险项、学习素材字段和要发布的任务
6. 点击 `确认发布选中任务`

发布完成后：

- 页面底部会保留快捷动作条
- 可以直接点击 `快捷看反馈` 或 `快捷去积分`

如果是背诵 / 朗读任务，发布时还应额外确认：

- 任务类型是否被识别成 `背诵` 或 `朗读`
- 标题、作者和正文是否已自动带出
- 是否需要对孩子隐藏标准原文

### 5.2 晚饭前检查完成情况

1. 点击底部 `反馈`
2. 先看日报摘要
3. 如果孩子做了默写，继续看听写结果
4. 需要复盘一周 / 一月走势时，再切趋势

适合关注的问题：

- 今天总共几项任务
- 还剩几项没完成
- 哪个学科拖后
- 听写是否已经批改完成

### 5.3 及时奖惩

1. 点击底部 `积分`
2. 选择奖励或扣分
3. 填写原因
4. 提交积分调整

当前屏会同时展示：

- 预计余额
- 当天流水
- 最近积分记录

这样不需要切屏对照。

### 5.4 晚上安排默写

1. 点击底部 `单词`
2. 设置清单名、语言、日期
3. 批量录入词项
4. 创建单词清单
5. 如需修正，直接在清单卡片里编辑

清单会自动同步到后端，Pad 端可直接使用。

## 6. 发布主屏的特殊说明

发布区是整个家长端最重要的主线，所以有三个固定设计：

- 顶部保留“解析 -> 审核 -> 发布”的阶段感
- 阶段卡点击后会直接切到对应子页面
- 发布区内部的 `范围 / 原文 / 审核 / 发布 / 拆分 / 任务 / 摘要 / 任务板` 已经拆成子页面，不再堆在一个长页面里
- 发布子菜单在手机视口下会保持粘性显示，滚动时不用回到顶部重新找入口

### 6.1 学习素材自动补全怎么用

当任务标题里出现 `背诵 / 朗读 / 古诗 / 诗词 / 跟读 / 诵读` 之类的关键词时，系统会把这类任务识别为需要学习素材的任务。

补全过程按下面优先级执行：

1. 家长手动输入优先
2. 如果家长没填，先从老师原文里自动抽取
3. 如果老师原文还不完整，再由 LLM 只补全文本缺口

这意味着家长在 `审核` 子页面中通常会直接看到：

- `reference_title`
- `reference_author`
- `reference_text`
- `analysis_mode`

常见场景：

- 老师原文里直接带了古诗全文：系统会自动抽取并带入审核卡
- 老师只写了“背诵《江畔独步寻花》”：如果配置了 LLM，系统会补出标题、作者和正文
- 家长自己改过标题 / 作者 / 正文：系统不再覆盖家长输入

默认策略：

- `背诵` 任务会默认对孩子隐藏标准正文，避免孩子直接照读
- `朗读` 任务可保留给孩子可见，便于对照跟读

### 6.2 审核背诵 / 朗读任务时看什么

在 `审核` 子页面，优先看四项：

1. 任务标题是否正确
2. 标题 / 作者是否匹配
3. 正文是否完整
4. `analysis_mode` 是否合理

一般判断方式：

- 古诗词短句、含朝代和作者信息的，通常应为 `classical_poem`
- 课文背诵通常应为 `text_recitation`
- 英语朗读通常应为 `english_reading`
- 普通朗读通常应为 `read_aloud`

底部动作条会根据阶段变化：

- 审核阶段：显示 `快捷发布`
- 发布完成阶段：显示 `快捷看反馈`、`快捷去积分`

录入原文阶段不再额外固定一个动作条，原因是：

- 录入页本身已经保留了解析按钮
- 手机底部还固定保留四大主屏导航
- 把固定动作条收口到“审核 / 发布完成”两个关键阶段，可以减少首屏被悬浮区域挤占

如果觉得页面信息多：

- 直接切换到对应子页面
- 不要在同一页里上下反复找区域
- 需要继续下一步时，优先使用底部动作条

## 7. 当前主屏置顶机制

家长端不再让四个大功能区一直按固定顺序铺开。

现在点击任何功能导航后，对应区域会被提到最前面。这样做的原因是：

- 家长通常不会四块一起看
- 更常见的是先处理一件当前最急的事
- 把当前区域置顶，比“长页面里滚来滚去”更接近原生 H5 的使用方式

当前实现还额外遵守一条约束：

- 不要求家长通过左右滑动才能看到发布主线、审核筛选、确认台、反馈切换或原因建议的完整内容

## 8. 桌面浏览器联调说明

即使在桌面浏览器中打开，家长端也会尽量保持手机壳体验：

- 页面宽度锁定为手机单列
- 底部导航固定在窗口底部
- 发布动作条固定在导航上方
- 不再以大屏多列控制台为主

因此联调时建议：

- 不需要把浏览器拉满宽度来观察布局
- 直接把它当成一台“放大版手机”使用

## 9. 验收关注点

验收家长端移动 H5 时，重点看下面几点：

1. 桌面打开时是否仍然是手机宽度，而不是 PC 多栏页面
2. 底部导航是否始终可点击
3. 当前主屏切换后是否会被置顶
4. 发布动作条是否会随阶段变化
5. 发布、反馈、积分、单词四个区域是否都能独立完成核心操作
6. 不同功能区切换后，原先填写的表单和已加载的数据是否保留
7. 发布步骤、筛选条、反馈切换和原因建议是否默认一次看全，不依赖横向拖动
8. 子页面切换时是否具有 App 风格左右过渡，而不是突然跳变或整页重排
