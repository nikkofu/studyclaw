# StudyClaw v0.3.2 一页摘要

发布日期：`2026-03-13`

## 这是什么

`v0.3.2` 是第一阶段稳定交付版 `v0.3.1` 之后的正式热修复版本。

它不扩功能范围，只收口两个必须修好的主路径：

- 家长端点击“去录入原文”后，必须看到输入框
- 孩子端点击“开始说话”后，不能再因为 STT 时序问题报错

## 给家长

- 现在在 `发布 -> 范围` 点击“去录入原文”，会直接进入 `原文` 子页面
- 不会再出现点了入口却仍停在起始页、找不到输入框的情况
- 家长端整体仍保持移动 H5 工位，不需要重新学习新的导航结构

## 给团队

- 这是一个热修复版，不是新的功能扩展版
- API 契约不变，三端事实源结构不变
- 本次主要修正的是家长发布路径和 Pad Web 语音输入路径
- 可以继续以 `v0.3.2` 作为下一阶段启动前的正式仓库基线

## 给 GitHub / 对外交付

`v0.3.2` 的意义是把第一阶段里两个影响真实使用体验的高频问题补平：

- 一个影响家长发作业
- 一个影响孩子用语音完成动作

因此它适合作为：

- 家长演示热修复版
- 团队协作热修复版
- GitHub 正式补丁标签

## 本版包含的关键修复

### 1. 家长端原文入口

- 修复空状态回退逻辑
- 点击“去录入原文”后直接进入输入页
- 原文录入、AI 解析、审核、发布主路径恢复连贯

### 2. Pad Web 语音助手

- 修复 `listen()` 返回值误判导致的 `Null bool` 报错
- 修复 interim transcript 后接 `done / notListening` 的 Web 时序问题
- 口令如“好了”“下一个”“数学订正好了”更稳定

## 验证状态

`v0.3.2` 已完成：

- release scope 检查
- Parent Web 自动化测试与构建
- Pad 自动化测试与构建
- 版本号与文档同步
- GitHub 主干同步与正式 tag / release

## 一句话结论

`v0.3.2` 代表的不是“又加了点东西”，而是把已经上线的真实主路径修到可放心交付。
