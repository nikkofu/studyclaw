# StudyClaw v0.3.2 补丁版使用说明

发布日期：`2026-03-13`

## 这份手册的定位

`v0.3.2` 是建立在 `v0.3.1` 正式交付基线上的补丁版本。

如果你需要完整的系统使用说明，先看：

- [docs/USER_MANUAL_V0.3.1.md](/Users/admin/Documents/WORK/ai/studyclaw/docs/USER_MANUAL_V0.3.1.md)
- [docs/PARENT_WEB_H5_MANUAL.md](/Users/admin/Documents/WORK/ai/studyclaw/docs/PARENT_WEB_H5_MANUAL.md)

这份文档只补充 `v0.3.2` 新修复的两条主路径：

1. 家长端点击“去录入原文”后，必须直接进入原文输入页。
2. 孩子端点击“开始说话”后，Pad 语音指令不能再因为 STT 返回值或 Web 收尾时序报错。

## 给家长

### 1. 发布作业时怎么进入原文输入框

在家长端底部切到 `发布` 主屏后：

1. 先在 `范围` 页选择孩子和日期。
2. 点击“去录入原文”。
3. 页面应直接切到 `原文` 子页面。
4. 此时应该立即看到原文输入框，可以直接粘贴老师消息。

`v0.3.2` 修复前，这一步有时会被错误带回 `范围` 页，导致家长看不到输入框。
`v0.3.2` 修复后，这条主路径已经恢复正常。

### 2. 推荐发布顺序

补丁版推荐仍按下面顺序操作：

1. `范围`
2. `原文`
3. `审核`
4. `发布`
5. `摘要 / 任务板`

如果要快速检查是否已经修好，只需要确认“点击去录入原文后，是否立刻能看到输入框”。

## 给孩子

### 1. 语音指令怎么用

在 Pad 的任务板或听写页：

1. 点击“开始说话”。
2. 浏览器第一次会请求麦克风权限，需要允许。
3. 对着设备说出简短指令。

当前推荐口令包括：

- `好了`
- `下一个`
- `继续`
- `Next`
- `数学订正好了`
- `一课一练做完了`

### 2. `v0.3.2` 修复了什么

补丁版重点修复了两个实际问题：

- 点击“开始说话”后，不再出现 `type 'Null' is not a bool in boolean expression`
- Web 端即使先收到中间识别结果，再收到 `done / notListening`，也能正常把 transcript 用起来，不会误判成失败

这意味着孩子说完“好了”“下一个”“数学订正好了”这一类短句后，系统更稳定地完成按钮等价交互。

家长端：

- API 契约不变

- 家长端移动 H5 的总体结构不变
- 孩子端正向鼓励逻辑不变

`v0.3.2` 不是新功能版，而是把两条高频主路径修到真正可交付：
家长能顺利进入原文录入，孩子能稳定使用“开始说话”完成语音指令。
