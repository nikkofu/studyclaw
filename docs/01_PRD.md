# StudyClaw 项目需求与可行性设计 (PRD)

## 1. 项目背景与愿景
**StudyClaw** 旨在打造一款专为上班族家庭的小朋友设计的“Agentic”（智能体驱动）日常学习与成长管理工具。它通过连接家长管理端与儿童使用端（Android Pad），利用兼容 OpenAI 协议的强大多模态大语言模型（LLM）实现智能解析、自动化任务布置、口语及听写交互陪伴。目标是释放家长的精力，并培养孩子自主学习的习惯。

## 2. 项目可行性与智能化体验深度思考

### 2.1 技术可行性
- **多模态与 LLM 协议广泛兼容性**: 采用 OpenAI 兼容层，系统可以轻松对接如 GPT-4o、Claude 3.5 Sonnet 等先进模型，实现文本、图像（Vision）和语音（TTS/ASR）的多模态闭环交互。
- **跨平台技术选型**:
  - Android Pad 端可使用 Flutter 或 React Native 实现高性能渲染及丰富的交互动效。
  - 管理端（前端网页或小程序）利用 React 或 Vue 实现响应式设计，便于家长随时操作。
  - **边缘感知与多传感器联动**: 利用 Pad 端的麦克风、环境光传感器和陀螺仪，为 AI 提供物理世界的上下文输入（如判断坐姿、用眼环境）。

### 2.2 深度 Agentic 生态 (The "Agentic" Shift)
不同于传统的被动式“指令-响应”软件，StudyClaw 是一个具有**主动性 (Proactivity)** 和 **自适应性 (Adaptability)** 的智能体集合：
- **主动观察与介入**: Agent 能够基于时间和传感器数据主动发声提醒休息，而非等待孩子去触发按钮。
- **长程情感记忆**: 从日常交互中提取性格特征与情绪状态并持久化，让“虚拟学伴”拥有连贯的记忆和真实的慰藉感。
- **自主复盘与升级**: 定期分析学情与交互反馈，自主调整沟通策略（Prompt 演进），真正实现“陪伴成长”。

### 2.2 商业与用户需求可行性
- **痛点明确**: 上班族家长通常在通勤或工作间隙通过微信获得学校群的作业通知，晚上下班后辅导孩子压力极大。StudyClaw 通过“家长一键转发/拍照 -> AI 解析归类 -> 孩子白天自主完成”的闭环，大幅度缓解了家长的焦虑。
- **用户粘性**: 结合奖励机制与正向反馈，不仅解决了任务分发问题，也提供了情感陪伴，符合新时代数智化家庭教育的需求。

## 3. 核心功能点 (MVP 阶段)

### 3.1 家长管理端 (后台/移动 Web)
* **作业解析与布置**: 支持文本粘贴、图片上传（OCR + LLM）。AI 自动提取出当天的语文、数学、英语等具体任务。
* **清单定制**: 自定义单词听写列表（导入课本词汇）、背诵列表、每日习惯打卡项。
* **亲子语音/文字留言**: 录制语音或输入文字，通过云端推送到孩子的设备端进行温馨播报。
* **数据看板与反馈**: 实时查看孩子的任务完成进度、得分记录及趋势分析。

### 3.2 孩子使用端 (Android Pad App)
* **任务清单 (To-Do List)**: 全天任务的图形化、大字号展示，带醒目的综合进度条。
* **交互式听写/背诵助手**: AI 语音播报单词，孩子在纸上写完后可拍照让 AI 判卷；或进行口述背诵，AI 提供纠错与提示。
* **备注与互动补充**: 允许孩子给任务添加语音备注或说明（“今天老师多留了半页算术”）。
* **奖励与成就中心**: 完成任务获取“爪印（Claws）”等虚拟积分，兑换家长设定的实体或虚拟奖励，拥有专属成长徽章墙。

## 4. 扩充与进阶功能点 (增值潜力)

为保证项目的长期生命力及趣味性，可以在后续逐步引入以下模块：

1. **AI 虚拟学伴角色 (Virtual Character / Avatar)**:
   - 孩子可以自定义一只专属宠物（例如小猫、小恐龙），随着学习任务的完成而升级、换装。所有的语音播报均由该虚拟伙伴发出。
2. **错题本与知识图谱自动生成**:
   - 拍照判定的作业、听写错误的单词自动进入个人的“专属错题库”，AI 会根据艾宾浩斯记忆曲线在随后的日子里自动生成复习任务。
3. **沉浸式番茄钟与专注自习室**:
   - 引入专注时长统计，模拟“太空旅行”等背景音效，孩子可以和其他线上的小伙伴进行匿名的学习时长比拼（轻社交）。
4. **数字家庭公约**:
   - 增加零花钱、做家务（如洗碗、拖地）的管理。完成某项家务后拍照上传，AI 初步认证，家长审核发放数字奖励。
5. **智能日记与总结反思**:
   - 每天晚上睡觉前，AI 引导孩子用语音进行 3 分钟的“今日回顾”，帮助孩子练习口头表达并自动生成格式化的成长日记给家长看。

## 5. 项目边界与安全性
* **数据隐私隔离**: 所有儿童的人脸、语音、家庭照片必须加密存储，严格遵循相关隐私法案（如类似 COPPA 的要求）。
* **防沉迷防误触设计**: Pad 界面交互应符合儿童人体工学原理，弱化容易导致误选的复杂菜单结构。护眼模式及晚间强制息屏机制。
* **LLM 隔离层（Safety Guardrails）**: 所有向大模型发出的 Prompts 都需通过过滤器，防范不宜内容和越狱攻击，确保回复的拟人化语言安全、积极、健康。
